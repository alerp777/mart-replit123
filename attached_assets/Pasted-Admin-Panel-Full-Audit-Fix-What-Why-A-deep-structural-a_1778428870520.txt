3. UI/UX & Responsiveness Issues
Z-Index conflicts:

PwaInstallBanner uses z-[9999] — sits above all modals and dialogs. When a modal is open and the PWA banner is shown simultaneously, the banner overlaps the modal overlay.
Mobile responsiveness (pages that break on small screens):

riders.tsx — stat cards and filterable table have no sm:/md: breakpoint classes; horizontal overflow is not handled.
transactions.tsx — wide ledger table has no horizontal scroll wrapper or column hiding on mobile.
orders/ sub-pages — OrdersFilterBar and OrderDetailDrawer lack mobile-specific layout adjustments.
Missing skeleton loaders (49 pages use isLoading with no skeleton):

The shared DataTable component has row-level skeleton rows — good.
Stat/KPI card sections at the top of pages (Riders, Users, Vendors, Finance) show nothing or a spinner while the first data fetch is in-flight, instead of skeleton placeholders.
Pages specifically missing top-section skeletons: riders.tsx, users.tsx, vendors.tsx, transactions.tsx, DepositRequests.tsx, Withdrawals.tsx, kyc.tsx, analytics.tsx.
4. Hidden Bugs
TypeScript errors (from pnpm tsc --noEmit):

File: src/pages/settings-payment.tsx
Count: 20 errors, all TS2322
Root cause: LucideIcon type is defined as FunctionComponent<SVGProps<SVGSVGElement>> in one @types/react version but the imported icons are ForwardRefExoticComponent<Omit<LucideProps, "ref"> & RefAttributes<SVGSVGElement>>. This happens because two incompatible copies of @types/react (v19) are resolved in the pnpm workspace — the VoidOrUndefinedOnly type is duplicated and the two copies are structurally incompatible.
Fix: Cast icon variables to React.ElementType or React.ComponentType<React.SVGProps<SVGSVGElement>> in settings-payment.tsx. Alternatively, declare a local type Icon = typeof SomeIcon and use it instead of the shared LucideIcon alias.
No React key warnings identified (static analysis; DataTable uses item IDs as keys).

No empty onClick handlers found — all visible buttons have handlers.

Console.log cleanup: 64 pages contain console.log statements that should be removed before production.

Steps
Fix settings-payment.tsx TypeScript errors — Replace all usages of the LucideIcon type alias in this file with React.ElementType or an inline type that matches ForwardRefExoticComponent. Run pnpm tsc --noEmit to confirm zero errors remain.

Fix PwaInstallBanner z-index — Lower z-[9999] to z-40 (or z-50 at most) so it sits below modal overlays which use z-50/z-[100]. Test that the banner still appears above the sidebar and top bar.

Add stat-card skeleton loaders — Create or reuse a StatCardSkeleton component (animated pulse placeholder matching the card dimensions). Apply it in the loading state of riders.tsx, users.tsx, vendors.tsx, transactions.tsx, DepositRequests.tsx, Withdrawals.tsx, kyc.tsx, and analytics.tsx.

Mobile layout fixes for data-heavy pages — Wrap the riders.tsx, transactions.tsx, and orders/OrdersFilterBar.tsx tables in a horizontal-scroll container (overflow-x-auto). Stack filter bar controls vertically on sm screens. Ensure OrderDetailDrawer occupies full-width on mobile.

Remove console.log from admin pages — Strip all bare console.log calls from files in artifacts/admin/src/pages/. Replace meaningful debug logs with a conditional check on import.meta.env.DEV if they need to stay for development.
